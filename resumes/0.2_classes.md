# Classes
## O que é uma classe? 
### Na sua aplicação, muitas vezes você vai encontrar vários objetos individuais que são semelhantes. Talvez existam milhares de bicicletas, todas da mesma marca e modelo. Cada bicicleta foi construída através dos mesmos diagramas e, além disso, compartilham os mesmos componentes. Nos termos da orientação a objetos, nós falamos que a sua bicicleta é uma instância da classe de objetos conhecidos como bicicleta. A classe é o diagrama do qual objetos individuais são criados.
## A classe Bicicleta a seguir é uma possível implementação de uma bicicleta
```java
    class Bicycle {
    
        int cadence = 0;
        int speed = 0;
        int gear = 1;
        
        void changeCadence(int newValue) {
            cadence = newValue;
        }
        
        void changeGear(int newValue) {
            gear = newValue;
        }
        
        void speedUp(int increment) {
            speed = speed + increment;
        }
        
        void speedDown(int decrement) {
            speed = speed - decrement;
        }
        
        void printStates() {
            IO.println("cadence:" +
            cadence  + " speed:" + 
            speed  + " gear:" + gear);
        }
    }
```
### A síntaxe da linguagem de programação Java talvez pareça nova para você, mas o design dessa classe é baseada na discussão prévia sobre o objeto bicicleta. Os campos cadence, speed, e gear representam o estado do objeto, e os métodos (changeCadence(), changeGear(), speedUp()) definem a interação com o mundo exterior.
### Talvez você tenha percebido que a classe Bicycle não contém um método main(), isso acontece por que não é um aplicação completa; é só um blueprint para bicicletas que talvez sejam usadas na aplicação. A responsabilidade de criar e usar novos objetos Bicicleta pertencem a algumas outras classes na sua aplicação.
## Aqui está uma Bicycledemo classe que cria duas Bicycle objetos e chama seus métodos.
```java
    class BicycleDemo {
        public static void main(String[] args) {
            // Create two different
            // Bicycle objects
            Bicycle bike1 = new Bicycle();
            Bicycle bike2 = new Bicycle();
            
            // invoke methos on
            // those objects  
            bike1.changeCadence(50);
            bike1.speedUp(10);
            bike1.changeGear(2);
            bike1.printStates();
            
            bike2.changeCadence(50);
            bike2.speedUp(10);
            bike2.changeGear(2);
            bike2.changeCadence(40);
            bike2.speedup(10);
            bike2.changeGear(3);
            bike2.printStates();
        }
    }
```
A saída desse teste imprime o estado final da cadência do pedal, a velocidade e a marcha para essas duas bicicletas:
```java
cadence: 50 speed:10 gear:2
cadence: 40 speed:20 gear:3
```
